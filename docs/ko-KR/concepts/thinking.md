# 사고 모드 (Thinking)

에이전트의 추론 깊이를 제어하는 기능입니다.

## 사고 모드란?

사고 모드는 에이전트가 응답 전에 "생각하는" 정도를 제어합니다:

- 높은 레벨: 더 깊은 추론, 더 나은 복잡한 문제 해결
- 낮은 레벨: 빠른 응답, 간단한 작업에 적합

## 사고 레벨

| 레벨      | 토큰 예산 | 용도                   |
| --------- | --------- | ---------------------- |
| `off`     | 0         | 사고 없음              |
| `minimal` | 1,024     | 기본적인 추론          |
| `low`     | 4,096     | 간단한 문제            |
| `medium`  | 10,240    | 일반적인 작업 (기본값) |
| `high`    | 32,768    | 복잡한 문제            |
| `xhigh`   | 65,536    | 매우 복잡한 추론       |

## 설정

### 기본 레벨

```json5
{
  agents: {
    defaults: {
      thinking: "medium",
    },
  },
}
```

### 에이전트별 레벨

```json5
{
  agents: {
    list: [
      {
        id: "main",
        thinking: "high",
      },
      {
        id: "quick",
        thinking: "low",
      },
    ],
  },
}
```

## 런타임 변경

### 채팅에서

```
/think high
/think medium
/think off
```

### 일시적 설정

```
/think high once
```

→ 다음 응답만 high로

## 자동 조정

### 복잡도 기반

```json5
{
  agents: {
    defaults: {
      thinking: {
        mode: "auto",
        min: "low",
        max: "high",
      },
    },
  },
}
```

에이전트가 질문 복잡도에 따라 자동 조정

## 비용 고려

사고 토큰도 API 비용에 포함됩니다:

| 레벨     | 추가 토큰 | 예상 비용 증가 |
| -------- | --------- | -------------- |
| `off`    | 0         | 0%             |
| `low`    | ~4K       | ~10%           |
| `medium` | ~10K      | ~25%           |
| `high`   | ~30K      | ~75%           |
| `xhigh`  | ~65K      | ~150%          |

## 사용 가이드

### 언제 높은 레벨?

- 복잡한 코딩 문제
- 디버깅
- 아키텍처 설계
- 수학/논리 문제

### 언제 낮은 레벨?

- 간단한 질문
- 빠른 응답 필요
- 비용 절약
- 간단한 작업

## 문제 해결

### 사고가 너무 오래 걸림

1. 레벨 낮추기
2. 타임아웃 설정 확인

### 품질이 떨어짐

1. 레벨 올리기
2. 프롬프트 개선
